Using a sample to estimate the population mean

Jeu 25 mars 2021
Category: Data Science

In this article we saw that the sampling distribution of the mean is normally distributed around the population mean. This property gives us a nice way of using a single sample to estimate population mean. We will see now how we can visualize this.

To start, let us re-create a nice sampling distribution of means from a population variable following a Lomax distribution with the following parameters:

  • the total population size
  • the size of each sample
  • the number of samples

Our outputs will be:

  • the population mean (population_mean)
  • the population standard deviation (population_std)
  • the means of all our samples (sample_means)
  • the standard deviations of all our samples (sample_stds)

def collect_samples_from_population(
    population_size, size_of_each_sample, number_of_samples):
    population = (np.random.pareto(3, population_size) + 1) * 2
    population_mean = population.mean()
    population_std = population.std()
    
    print("Population mean: {:.3f} ".format(population_mean))
    print("Population standard deviation: {:.3f}".format(population_std))
    
    sample_means = list()
    sample_stds = list()
    for i in range(0, number_of_samples):
        sample = np.random.choice(population, size_of_each_sample, replace=False)
        sample_means.append(np.mean(sample))
        sample_stds.append(np.std(sample))
        del sample
    del population
        
    return population_mean, population_std, sample_means, sample_stds

def plot_sample_means(
    sample_means, population_mean, xmin=None, xmax=None, ymax=None):
    fig, ax = plt.subplots(figsize=(10, 5))
    
    # the weights are used to display percentages of samples
    n = len(sample_means)
    weights_pc = np.array([1.]*n) / n
    
    bins_n, bins_lower, patches = ax.hist(
        sample_means, bins=100, weights=weights_pc,
        # label="Means of the samples",
    )

    ax.axvline(
        population_mean, color="r", linestyle="solid", linewidth=1,
        label="Population mean: {:.3f}".format(round(population_mean, 3))
    )
    
    if xmin is None:
        xmin = ax.get_xlim()[0]
    if xmax is None:
        xmax = ax.get_xlim()[1]
    if ymax is None:
        ymax = ax.get_ylim()[1]

    # formatting 
    y_ticks_loc = ax.get_yticks().tolist()
    ax.yaxis.set_major_locator(mticker.FixedLocator(y_ticks_loc))
    ax.set_yticklabels(["{:,.1%}".format(y) for y in y_ticks_loc])
    ax.set_xlim((xmin, xmax))
    ax.set_xlabel("Sample means", fontsize=12)
    ax.set_ylim((0, ymax) )
    ax.set_ylabel("Percentage of samples in bin", fontsize=12)
    ax.legend(loc="upper right")

    return fig, ax, bins_n, bins_lower, patches

Good, it still looks normally distributed - bell shaped - and centered around the population mean (it is not perfectly normally distributed, but we will come back to that later).

So what now ?¶

Well now we need to ask ourselves to following question:

When I have a single sample, how do I estimate the population_mean and population_std_dev?

Our main issue here is that our sample is somewhere in the blue bell, but we are not sure where exactly. We are going to address that incertainty by building an interval around our single sample mean in which we believe the population mean is likely to be (I would like to emphazise that this is far from perfect, but this the best we can do given the circonstances).

Let us keep the following elements in mind.

  • I have a collected a single sample and written down its mean and standard deviation.
  • Sampling is a random process so if I draw another sample mean and standard deviation will be different.
  • Sample means are normally distributed.
  • I want to have a value range for my population mean.

Probability intervals¶

A nice property of normal distributions is that 95.45% of its values lies within the 2 first standard deviations around the population_mean (more about this here). Let us see if we can work something around that. We will want to know the standard deviation of the sampling distribution of means, that give us a measure of how much the sample means are spread around the population_mean. We will compute:

  • the mean of the sampling distribution of means (sdom_mean)
  • the standard deviation of the sampling distribution of means (sdom_std)
  • see how many of the sample means are within 2 sdom_stdof the sdpm_mean

We can then color this interval on our histogram.

In [6]:
def sdom_and_95pc_rule(sample_means):
    sdom_mean = np.mean(sample_means)
    sdom_std = np.std(sample_means)
    
    print("SDM mean: {:.6f} ".format(sdom_mean))
    print("SDM standard deviation: {:.6f}".format(sdom_std))

    means_in_2_std_dev = [
        m for m in sample_means if (
            m < (sdom_mean + 2.*sdom_std) and m > (sdom_mean - 2.*sdom_std)
        )]
    text = "Percentage of means inside a 2x std deviation interval: {:.2%}"
    print(text.format(len(means_in_2_std_dev) / len(sample_means)))
    
    return sdom_mean, sdom_std

def add_proba_range_to_plot(
    fig, ax, bins_n, bins_lower, patches, selected_mean, selected_range):
    central_bin = bins_lower[bins_lower < selected_mean].max()
    central_index = np.where(bins_lower == central_bin)[0][0]

    lower_mean = selected_mean - selected_range
    lower_bin = bins_lower[bins_lower < (lower_mean)].max()
    lower_index = np.where(bins_lower == lower_bin)[0][0]

    upper_mean = selected_mean + selected_range    
    upper_bin = bins_lower[bins_lower < (upper_mean)].max()
    upper_index = np.where(bins_lower == upper_bin)[0][0]
    
    for i in range(lower_index, upper_index):
        if i == central_index:
            patches[i].set_color("orange")
        else:
            patches[i].set_color("navajowhite")
        
        n_in_range = bins_n[lower_index:upper_index].sum()
        p_in_range = n_in_range / bins_n.sum()
    
    # here I add extra information to the legend
    ax.plot([], [], "orange", 
            label="All samples with mean between {:.3f} and {:.3f}".format(
                lower_bin, upper_bin))
    ax.plot([], [], " ", 
            label="Number of samples in range: {:,.2%}".format(
                p_in_range))
    ax.legend(loc="upper right")
    
    return fig, ax, p_in_range

We are using bins for our histogram, so it is normal not to find the exact same percentage as with the computation above, but this is close enough to 95%. This interval is a probability interval. Picking up new samples we have a 95% chance to have a sample mean inside this interval. This is a usefull step, but it does not solve our problem as the sdom_mean and sdom_std are unknow when we have only a single sample. We need to turn the solution around.

Confidence intervals¶

Something we can do around our single sample mean is to build an interval in which we are confident that the population mean is, a confidence interval. The idea is that 95% of our confidence intervals would actually include the population_mean. It also means that we have a 5% chance of having collected a sample for which the confident interval does not include the population_mean.

We differentiate probability interval and confidence interval as they do not cover the same thing exactly. For probability intervals, we have no sample yet, we forecast the results of a random process. For confidence interval we already have the result of the random process (our sample), so we retrospect the chance to be in the right range.

We can follow the same way as before. We will just replace the terms. We don't have the sdom_mean nor the population_mean, so we will replace it with our single sample mean. So by design, our confidence interval will be centered around it. So if we knew the sdom_std, we could just do the same as for the probability interval at 95,45%. Let us try this out.

def test_95pc_confidence(sample_means):
    c = 0
    for i in range(0, len(sample_means)):
        if (population_mean < (sample_means[i] + 2.*sdom_std) \
            and population_mean > (sample_means[i] - 2.*sdom_std)):
            c = c + 1
    text = """Percentage of samples for which the population mean 
is inside the confidence interval {:.2%}"""
    print(text.format(c/len(sample_means)))

Alright, using sample_mean and 2 * sdom_std gives us a 95% confidence interval. If we can replace sdom_std, we have all we need. It happends that sdom_std (the standard deviation of sampling distribution of means = SEM or standard error of the mean) is equal to population_std divided by the square root of the sample_size. This can be demonstrated mathematically, but we will just test it here.

def test_95pc_confidence_from_sample_mean_and_std(
    size_of_each_sample, sample_means, sample_stds):
    c = 0
    for i in range(0, len(sample_means)):
        if (population_mean < 
            (sample_means[i] + 2.*(sample_stds[i] / (size_of_each_sample**(1/2)))) 
            and population_mean > 
            (sample_means[i] - 2.*(sample_stds[i] / (size_of_each_sample**(1/2))))):
            c = c + 1
    text = """Percentage of samples for which the population mean 
is inside the confidence interval {:.2%}"""
    print(text.format(c/len(sample_means)))

Being more confident of confidence intervals: edge cases¶

Now let us dive into several elements we overlooked.

Normality¶

Our original population distribution is not normal at all: asymetric, heavy tail on the left and long tail on the right. That reflects the same way in the sampling distribution of means. We see it when comparing the left and right side at the top and on the tails.

In [19]:
def add_asymetry_to_plot(
    fig, ax, bins_n, bins_lower, patches, sdom_mean, sdom_std):
    import matplotlib.patches as mpatches
    
    min_y, max_y = ax.get_ylim()
    formatting = {"linewidth": 1, "edgecolor":"k", "facecolor": "none"}
    
    # top
    top_x_delta = 0.1
    top_x_min = sdom_mean - (top_x_delta / 2)
    top_y_delta = 0.01
    top_y_min = max_y - top_y_delta
    top_rectangle = mpatches.Rectangle(
        (top_x_min, top_y_min), top_x_delta, top_y_delta, **formatting
    )

    # bottom (b)
    b_x_delta = 0.2
    b_y_delta = 0.002
    b_y_min = min_y

    # bottom right (br)
    br_x_min = sdom_mean + (2*sdom_std) 
    br_rectangle = mpatches.Rectangle(
        (br_x_min, b_y_min), b_x_delta, b_y_delta, **formatting
    )

    # bottom left (bl)
    bl_x_min = sdom_mean - (2*sdom_std) - b_x_delta
    bl_rectangle = mpatches.Rectangle(
        (bl_x_min, b_y_min), b_x_delta, b_y_delta, **formatting
    )
    
    # adding to plot
    ax.add_patch(top_rectangle)
    ax.add_patch(br_rectangle)
    ax.add_patch(bl_rectangle)
    
    return fig, ax

Because of those differences with a normal distribution, the underlying values (following a Lomax in our case) are often trimmed from outliers (1) or transformed to become a normal distribution (2). This is especially necessary for small sample size.

Sample size, z-score, t-score...¶

A second very important point is the sample size. We used a big sample size, but a lot of the science cannot aford that type of sample size. So how does a small sample size affects our confidence interval?

Because it impacts the normality of the sampling distribution of means, it impacts the coefficient we want use to multiply the sdom_std. The 2 by which we multiply the standard deviation to get 95.45% is a called a z-value. z-values are taken for a standard normal distribution (mean = 0 and standard deviation = 1). For small samples, it is not good enough. We need to use t-values. They are very close from z-values for big samples (30 < x < 10% population) but differ quite a bit for smalled samples.
